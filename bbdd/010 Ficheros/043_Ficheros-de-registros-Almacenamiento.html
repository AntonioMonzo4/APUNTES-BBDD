Ficheros de registros - Almacenamiento

Formatos de almacenamiento

Para facilitar el intercambio de este tipo de ficheros entre distintas aplicaciones, se utilizan unos formatos estándar, que son el CSV, JSON y XML.

- El formato CSV es muy utilizado para almacenar tablas de hojas de cálculo, como el Excel o el Calc.
- Los formatos JSON y XML son muy utilizados para intercambiar datos entre aplicaciones web, y en la creación de ficheros de configuración de aplicaciones.

CSV: Comma-Separated Values

En estos archivos las columnas quedan definidas por coma (,) o punto y coma (;), mientras que cada fila se define mediante una línea adicional en el texto.

Si se utiliza como delimitador el tabulador, se suelen denominar TSV

Continente;País;Capital
África;Angola;Luanda
América del Norte;Estados Unidos;Washington DC
América Central;México;Ciudad de México
América del Sur;Brasil;Brasilia
Europa;España;Madrid
Europa;Alemaña;Berlín

JSON: JavaScript Object Notation

Los ficheros JSON originalmente se utilizaban para definir objetos en JavaScript, pues siguen la misma notación:
{ atributo1: valor1, atributo2: valor2, ...} donde valor puede ser a su vez, otro objeto JSON.

{
  "localidad 1": {
      "Continente": "África",
      "País": "Angola",

  "localidad 3": {

XML (eXtensible Markup Language)

El XML es un lenguaje de marcas

<Capitales>
  <Localidad número="1">
      <Continente>África</Continente>
      <País>Angola</País>
      <Capital>Luanda</Capital>
  </Localidad>
  <Localidad número="2">
