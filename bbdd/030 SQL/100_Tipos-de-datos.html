Tipos de Datos (mySQL)

Todos los valores de un mismo atributo siempre deben seguir un mismo formato, es decir, deben ser de un mismo tipo. Ahora debemos pensar en cómo vamos a representar cada uno de esos valores.

Por ejemplo, un DNI siempre se representa como un conjunto de dígitos y una letra al final.

Si tuviéramos que clasificarlo diríamos que es un conjunto de caracteres alfanuméricos.

¿Y el Sueldo? para nosotros sería un número entero con dos decimales.

Al crear la relación (tabla) decidimos qué conjunto de datos deberá almacenar en las filas de los distintos atributos, es decir: a cada atributo debemos asignarle un tipo de dato.

Entonces, para cada atributo de una tabla debemos tomar dos decisiones:
- Qué nombre va a tener.
- Qué tipo de datos va a almacenar.

Existen distintas formas de nombrar los tipos de datos dependiendo del lenguaje utilizado (C, Pascal, java, ...) y del SGBD (MySQL, SQL Server, Oracle DB, Access, ...).

Los tipos utilizados en MySQL 8.0 son:

Texto

CHAR(size)A FIXED length string (can contain letters, numbers, and special characters). The size parameter specifies the column length in characters - can be from 0 to 255. Default is 1
VARCHAR(size)A VARIABLE length string (can contain letters, numbers, and special characters). The size parameter specifies the maximum string length in characters - can be from 0 to 65535
BINARY(size)Equal to CHAR(), but stores binary byte strings. The size parameter specifies the column length in bytes. Default is 1
VARBINARY(size)Equal to VARCHAR(), but stores binary byte strings. The size parameter specifies the maximum column length in bytes.
TINYBLOBFor BLOBs (Binary Large Objects). Max length: 255 bytes
TINYTEXTHolds a string with a maximum length of 255 characters
TEXT(size)Holds a string with a maximum length of 65,535 bytes
BLOB(size)For BLOBs (Binary Large Objects). Holds up to 65,535 bytes of data
MEDIUMTEXTHolds a string with a maximum length of 16,777,215 characters
MEDIUMBLOBFor BLOBs (Binary Large Objects). Holds up to 16,777,215 bytes of data
LONGTEXTHolds a string with a maximum length of 4,294,967,295 characters
LONGBLOBFor BLOBs (Binary Large Objects). Holds up to 4,294,967,295 bytes of data
ENUM(val1, val2, val3, ...)A string that can have only one value, chosen from a list of possible values. If a value is inserted that is not in the list, a blank value will be inserted. The values are sorted in the order you enter them.
- You can list up to 65535 values in an ENUM list.
SET(val1, val2, val3, ...)A string that can have 0 or more values, chosen from a list of possible values.
-You can list up to 64 values in a SET list.

Numérico

BIT(size)The number of bits per value is specified in size. The size parameter can hold a value from 1 to 64. The default value for size is 1.
TINYINT(size)Signed range is from -128 to 127. Unsigned range is from 0 to 255. The size parameter specifies the maximum display width (which is 255)
BOOL
BOOLEAN		Zero is considered as false, nonzero values are considered as true.
SMALLINT(size)Signed range is from -32768 to 32767. Unsigned range is from 0 to 65535.
The size parameter specifies the maximum display width (which is 255)
MEDIUMINT(size)Signed range is from -8388608 to 8388607. Unsigned range is from 0 to 16777215.
The size parameter specifies the maximum display width (which is 255)
INT(size)
INTEGER(size)		Signed range is from -2147483648 to 2147483647.
Unsigned range is from 0 to 4294967295.
The size parameter specifies the maximum display width (which is 255)
BIGINT(size)Signed range is from -9223372036854775808 to 9223372036854775807.
Unsigned range is from 0 to 18446744073709551615.
The size parameter specifies the maximum display width (which is 255)
FLOAT(p)MySQL uses the p value to determine whether to use FLOAT or DOUBLE for the resulting data type. If p is from 0 to 24, the data type becomes FLOAT(). If p is from 25 to 53, the data type becomes DOUBLE()
DOUBLE(size, d)The total number of digits is specified in size. The number of digits after the decimal point is specified in the d parameter
DEC(size, d)
DECIMAL(size, d)		An exact fixed-point number. The total number of digits is specified in size. The number of digits after the decimal point is specified in the d parameter.
- The maximum number for size is 65. The maximum number for d is 30.
- The default value for size is 10. The default value for d is 0.

Note: Todos los tipos numéricos admiten las opciones UNSIGNED or ZEROFILL. If you add the UNSIGNED option, MySQL disallows negative values for the column. If you add the ZEROFILL option, MySQL automatically also adds the UNSIGNED attribute to the column.

Fecha y hora

DATEYYYY-MM-DD. The supported range is from '1000-01-01' to '9999-12-31'
DATETIME(fsp)YYYY-MM-DD hh:mm:ss. The supported range is from '1000-01-01 00:00:00' to '9999-12-31 23:59:59'. Adding DEFAULT and ON UPDATE in the column definition to get automatic initialization and updating to the current date and time.
TIMESTAMP(fsp)YYYY-MM-DD hh:mm:ss. The supported range is from '1970-01-01 00:00:01' UTC to '2038-01-09 03:14:07' UTC. Automatic initialization and updating to the current date and time can be specified using DEFAULT CURRENT_TIMESTAMP and ON UPDATE CURRENT_TIMESTAMP in the column definition.
TIMESTAMP values are stored as the number of seconds since the Unix epoch ('1970-01-01 00:00:00' UTC).
TIME(fsp)hh:mm:ss. The supported range is from '-838:59:59' to '838:59:59'
YEARA year in four-digit format. Values allowed in four-digit, from 1901 to 2155, and 0000.
