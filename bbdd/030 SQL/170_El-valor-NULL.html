El valor NULL

¿Qué sucede si al guardar un registro hay algún campo del que en ese momento no tengo su valor, o no necesito guardarlo porque no se aplica?

Independientemente del dominio al que pertenezca un campo, un campo puede tomar un valor especial denominado NULL que designa ausencia de dato.

No es lo mismo guardar un 0 o un espacio en blanco que un NULL, el no incluir nada en un campo es incluir un NULL, que significa "ausencia de valor", bien porque no se aplica o porque no lo tengo.

La lógica del NULO

¿Cómo se resuelven las operaciones lógicas sobre este valor?

En la lógica booleana tenemos los valores true y false, pero un valor NULL no es ni verdadero ni falso:

▸ Con el operador OR, basta que un operando sea true para que el resultado sea true, pero con NULL no sabemos si es true o false:

true OR NULL es true
false OR NULL es NULL

▸ Con el operador AND, basta que un operando sea false para que el resultado sea false, pero con NULL no sabemos si es false o true:

false AND NULL es false
true AND NULL es NULL

▸ Con NOT es imposible saber si el resultado será true o false:

NOT NULL es NULL

operador IS NULL

Si necesitamos otro resultado, debemos definir la lógica con este valor, para ello tenemos el operador campo IS NULL que devuelve true cuando el campo es NULO.

Por ejemplo, para mostrar todos los registros de una tabla que en uno de sus campos tienen un nulo:

 SELECT * FROM clientes WHERE telefono IS NULL;

 SELECT * FROM alumnos WHERE edad < 18 OR edad IS NULL;
