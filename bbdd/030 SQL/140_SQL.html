SQL: Structured Query Language

El lenguaje SQL es uno de los más utilizados en la industria informática en todos los tiempos.

SQL es un lenguaje declarativo: lo más importante es definir "qué se desea hacer", y no cómo hacerlo.
De esto último ya se encarga el motor de la base de datos.

El motor por defecto del SGBD de mariaDB es innoDB. Para ver qué
otros motores tenemos disponibles y sus características:
   SHOW ENGINES;

SQL es un lenguaje estándar, aunque cada base de datos tiene sus peculiaridades. Es conveniente revisar siempre la documentación del SGBD con el que trabajamos, ya que puede tener alguna característica que no cumpla el estándar.

Comandos

Se establecen tres tipos de comandos, según su uso:

1. Comandos DDL: Lenguaje de Definición de Datos.

CREATECrea nuevas bases de datos, tablas, vistas índices, usuarios, procedimientos, triggers.
DROPElimina bases de datos, tablas, vistas, e índices usuarios, procedimientos, triggers.
ALTERModifica la estructura de tablas, vistas, usuarios, procedimientos, triggers.

2. Comandos DML. Lenguaje de Manipulación de Datos.

SELECTConsulta filas que satisfagan un criterio determinado
INSERTIntroduce nuevas filas en una tabla
UPDATEModifica valores de los campos
DELETEElimina filas

3. Comandos DCL. Lenguaje de Control de Datos.

GRANTConcede permisos a usuarios sobre tablas o bases de datos
REVOKEElimina permisos previamente concedidos con GRANT

Cláusulas

Condiciones o criterios, son palabras especiales que permiten modificar el funcionamiento de un comando.

Por ejemplo, las cláusulas utilizadas con el comando SELECT:

FROMEspecifica la tabla de la que se usará en la instrucción SQL
WHEREEspecifica las condiciones que deben reunir las filas a tratar
GROUP BYSepara las filas seleccionadas en grupos específicos
HAVINGExpresa la condición que debe satisfacer cada grupo
ORDER BYOrdena las filas seleccionadas de acuerdo a un criterio específico
LIMIT(MySQL) Limita el número de filas a mostrar en la consulta

Otras cláusulas de SELECT estechamente vinculadas con el álgebra relacional son:

UNIONMuestra los resultados de varias consultas, sin repeticiones
INTERSECTIONMuestra las filas que tienen en común varias consultas
JOINRealiza consultas relacionando varias tablas

Operadores

Permiten crear expresiones complejas. Pueden ser:
- aritméticos (+, -, *, /, %) o
- lógicos (<, <=, =, >=, >, <>, AND, OR, NOT, LIKE, BETWEEN, IN, EXIST, ANY, ALL).

ANDEvalúa dos condiciones y devuelve un valor de verdad sólo si ambas son ciertas.
OREvalúa dos condiciones y devuelve un valor de verdad si alguna de las dos es cierta.
NOTDevuelve el valor contrario de la expresión.

Operadores de comparación

BETWEENEspecifica un intervalo de valores.
LIKERealiza comparaciones con metacaracteres
INSelecciona filas con cierto campo dentro de un conjunto de datos

Funciones

Las funciones nos permiten realizar operaciones sobre valores, que pueden ser los campos de una tabla. Esos valores deben pasarse como argumento a la función:
nombreFuncion(arg1, arg2, ...)

Funcines mariadb

  • de texto
  • de fecha/hora
  • de agregación
  • numéricas
  • de control de flujo

Las funciones de agregado hacen un "resumen" de valores de campos, y se suelen utilizarse en combinación con HAVING y/o GROUP BY.

FunciónDescripción
AVGCalcula el promedio de los valores de un campo determinado.
COUNTDevuelve el número de filas de la selección.
SUMDevuelve la suma de todos los valores de un campo determinado.
MAXDevuelve el valor más alto de un campo determinado.
MINDevuelve el valor mínimo de un campo determinado.
